# 穿搭灵感匹配工具 — 极速 MVP 计划（1-2 天上线）

> 项目代号：**StyleMate（搭搭）**
> 形态：微信小程序
> 目标：**1-2 天内跑通核心链路，上线验证市场**
> 策略：砍掉一切非必要功能，全程 AI 驱动开发，只保留"拍衣服 → 出搭配"

---

## 一、核心思路：只做一件事做到极致

### 1.1 极简用户旅程（唯一链路）

```
用户打开小程序
    │
    ▼
拍照 / 上传衣物照片（1-N 张）
    │
    ▼
AI 自动识别衣物（类型、颜色、风格）
    │
    ▼
AI 生成 3 套搭配方案（文字描述 + 搭配卡片 + 参考图）
    │
    ▼
保存搭配图到相册 → 发小红书 / 朋友圈
```

### 1.2 MVP 砍什么、留什么

| 功能 | 完整版 | ✅ MVP 保留 | ❌ MVP 砍掉 |
|------|--------|------------|------------|
| 微信登录 | 完整用户系统 | ✅ 仅静默登录获取 openid | ❌ 资料设置、风格偏好 |
| 数字衣橱 | 完整 CRUD + 分类 | ✅ 拍照上传 + AI 识别 + 列表展示 | ❌ 编辑、筛选、统计 |
| AI 搭配 | 天气/场合/个性化 | ✅ 从已上传衣物中推荐搭配 | ❌ 天气联动、场合选择 |
| 搭配展示 | 虚拟试穿 + 参考图 | ✅ 衣物拼图卡片 + AI 文字方案 | ❌ 虚拟试穿 |
| 分享 | 多渠道 + 社区 | ✅ 保存图片到相册 | ❌ 社区、小程序码 |
| 种草 | 商品推荐 + CPS | ❌ 全部砍掉 | ❌ 后续再加 |
| 会员 | 付费体系 | ❌ 全部砍掉 | ❌ 后续再加 |

┌──▼───┐  ┌──▼──────────────┐
│SQLite│  │  AI 服务（直接调） │
│本地DB │  │  • 衣物识别: 多模态LLM │
└──────┘  │  • 搭配推荐: LLM     │
          │  • 抠图: rembg       │
          └──────┬──────────────┘
                 │
          ┌──────▼──────┐
          │  文件存储     │
          │  本地/COS    │
          └─────────────┘

### 2.2 极简技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| **前端** | 微信小程序原生 | 最快，无需额外框架 |
| **后端** | FastAPI（单文件起步） | 你最熟，开发最快 |
| **数据库** | SQLite | 零配置，MVP 够用，后续可迁 PostgreSQL |
| **图片存储** | 本地文件系统 + 静态服务 | MVP 不需要 COS，省时间 |
| **AI 识别** | 通义千问-VL / GPT-4o | 一个 API 搞定衣物识别 |
| **AI 搭配** | DeepSeek / GPT-4o | 一个 Prompt 出搭配方案 |
| **抠图** | rembg（本地）| 离线运行，免费，效果好 |
| **部署** | 腾讯云轻量 / CloudStudio | 一键部署 |

## 三、后端设计（极简 3 个接口）

### 3.1 项目结构

```
backend/
├── app.py                    # FastAPI 主文件（所有接口）
├── ai_service.py             # AI 识别 + 推荐（封装 LLM 调用）
├── image_service.py          # 图片处理（抠图 + 搭配卡片生成）
├── models.py                 # SQLite 数据模型（SQLModel）
├── config.py                 # 配置（API Key 等）
├── uploads/                  # 上传的图片存储
├── processed/                # 抠图后的图片
├── cards/                    # 生成的搭配卡片
├── requirements.txt
└── Dockerfile
```

### 3.2 数据模型（SQLModel，极简）

```python
from sqlmodel import SQLModel, Field
from datetime import datetime
from typing import Optional

class User(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    openid: str = Field(unique=True, index=True)
    created_at: datetime = Field(default_factory=datetime.now)

class ClothingItem(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    user_id: int = Field(index=True)
    image_url: str                          # 原图路径
    thumbnail_url: Optional[str] = None     # 抠图后路径
    category: str                           # 上衣/裤装/裙装/外套/鞋
    color: str                              # 主色调
    style: str                              # 风格标签
    ai_description: str                     # AI 描述
    created_at: datetime = Field(default_factory=datetime.now)

class Outfit(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    user_id: int = Field(index=True)
    item_ids: str                           # 逗号分隔的衣物 ID
    description: str                        # AI 搭配说明
    card_url: Optional[str] = None          # 搭配卡片图 URL
    created_at: datetime = Field(default_factory=datetime.now)
```

### 3.3 核心接口（只有 3 个）

```
POST  /api/upload          # 上传衣物 → AI 识别 → 入库 → 返回识别结果
GET   /api/wardrobe        # 获取用户衣橱列表
POST  /api/recommend       # 从衣橱选衣物 → AI 出搭配 → 生成卡片 → 返回
```

#### 接口 1：上传衣物

```
POST /api/upload
Content-Type: multipart/form-data

请求：
  - image: File（衣物照片）
  - openid: str（用户标识）

处理流程：
  1. 保存原图 → uploads/
  2. rembg 抠图去背景 → processed/
  3. 调用多模态 LLM 识别衣物属性
  4. 存入 SQLite
  5. 返回识别结果

响应：
{
    "id": 1,
    "image_url": "/static/uploads/xxx.jpg",
    "thumbnail_url": "/static/processed/xxx.png",
    "category": "上衣",
    "color": "白色",
    "style": "简约休闲",
    "ai_description": "白色圆领短袖T恤，纯棉材质，适合春夏"
}
```

#### 接口 2：获取衣橱

```
GET /api/wardrobe?openid=xxx

响应：
{
    "items": [
        { "id": 1, "thumbnail_url": "...", "category": "上衣", "color": "白色", ... },
        { "id": 2, "thumbnail_url": "...", "category": "裤装", "color": "蓝色", ... }
    ],
    "total": 2
}
```

#### 接口 3：AI 搭配推荐

```
POST /api/recommend
Content-Type: application/json

请求：
{
    "openid": "xxx",
    "item_ids": [1, 2, 3, 5]    // 可选，指定用哪些衣物；不传则用全部
}

处理流程：
  1. 查询用户衣橱（或指定衣物）
  2. 构建 Prompt，调用 LLM 生成 3 套搭配
  3. 用 Pillow 生成搭配拼图卡片
  4. 存入数据库
  5. 返回搭配方案

响应：
{
    "outfits": [
        {
            "id": 1,
            "items": [
                { "id": 1, "thumbnail_url": "...", "category": "上衣" },
                { "id": 3, "thumbnail_url": "...", "category": "裤装" }
            ],
            "description": "简约通勤风：白色T恤搭配深蓝直筒牛仔裤...",
            "style_tags": ["简约", "通勤"],
            "card_url": "/static/cards/outfit_1.png"
        },
        ...
    ]
}
```

## 四、AI Prompt 设计

### 4.1 衣物识别 Prompt

```
你是一位专业的服装分析师。请分析这张衣物图片，返回以下 JSON 格式信息：

{
    "category": "上衣/裤装/裙装/外套/鞋/包/配饰",
    "sub_category": "具体类型如T恤/衬衫/牛仔裤/连衣裙等",
    "color": "主色调",
    "style": "风格标签如休闲/正式/运动/甜美/学院/新中式等",
    "season": "适合季节",
    "description": "一句话描述这件衣物"
}

只返回 JSON，不要其他文字。
```

### 4.2 搭配推荐 Prompt

```
你是小红书上最火的穿搭博主，精通各种流行搭配风格。

## 用户衣橱里有这些衣物：
{wardrobe_list}

## 任务
请从这些衣物中搭配出 3 套穿搭方案。

## 要求
1. 每套至少包含上装 + 下装（有合适的外套/鞋/配饰也可加入）
2. 颜色要协调
3. 风格要统一
4. 给出搭配理由（用小红书博主的语气，活泼有趣）
5. 每套给 1-2 个风格标签

## 输出格式（严格 JSON）
{
    "outfits": [
        {
            "items": [衣物ID列表],
            "style_tags": ["标签1", "标签2"],
            "description": "搭配说明，小红书风格文案，50字以内",
            "reason": "为什么这样搭好看，30字以内"
        }
    ]
}

只返回 JSON，不要其他文字。
```

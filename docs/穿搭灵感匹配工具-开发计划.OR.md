# 穿搭灵感匹配工具 — 小程序开发计划

> 项目代号：**StyleMate（搭搭）**
> 形态：微信小程序
> 目标用户：18-35 岁女性，日常有穿搭困扰，活跃于小红书等社交平台

---

## 一、项目总览

### 1.1 核心痛点

"我有一堆衣服，但每天不知道怎么搭。"

### 1.2 产品愿景

打造一个**轻量级、社交化、本土审美**的 AI 穿搭助手小程序，让用户拍照建衣橱 → AI 智能搭配 → 一键种草分享，形成完整闭环。

### 1.3 差异化定位

| 维度 | 竞品（Stylebook / Whering） | 我们（StyleMate） |
|------|---------------------------|-------------------|
| 形态 | 重 App，需下载 | 微信小程序，即用即走 |
| 审美 | 欧美风格为主 | 本土审美（辣妹/学院/新中式等） |
| 社交 | 无社交属性 | 搭配结果一键分享小红书 |
| 语言 | 英文 | 中文原生 |
| 购买 | 无闭环 | 种草推荐 + 购买跳转 |

## 二、技术架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                  微信小程序（前端）                 │
│  Taro / 原生微信小程序 + TailwindCSS              │
└──────────────────────┬──────────────────────────┘
                       │ HTTPS
┌──────────────────────▼──────────────────────────┐
│                 后端 API 层                       │
│          Python FastAPI + Uvicorn                │
│  ┌───────────┬──────────┬──────────────────┐     │
│  │ 用户服务   │ 衣橱服务  │ 搭配推荐服务     │     │
│  └───────────┴──────────┴──────────────────┘     │
└──────┬──────────┬───────────┬───────────────────┘
       │          │           │
┌──────▼───┐ ┌───▼────┐ ┌───▼──────────────┐
│ PostgreSQL│ │ Redis  │ │   AI 服务层       │
│ (主数据库) │ │ (缓存)  │ │                  │
└──────────┘ └────────┘ │ • 衣物识别(CV)    │
                        │ • 搭配推荐(LLM)   │
                        │ • 图片合成(预留)   │
                        └──────┬───────────┘
                               │
                        ┌──────▼───────────┐
                        │  对象存储 (COS)    │
                        │  衣物图片/生成图    │
                        └──────────────────┘
```

### 2.2 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| **前端** | 微信小程序原生 / Taro 3 | 原生体验好，Taro 可跨端复用 |
| **后端** | Python FastAPI | 你熟悉，性能好，AI 生态完善 |
| **数据库** | PostgreSQL + pgvector | 结构化数据 + 向量检索（衣物相似度） |
| **缓存** | Redis | 搭配结果缓存、限流、session |
| **对象存储** | 腾讯云 COS | 衣物图片存储，CDN 加速 |
| **AI - 衣物识别** | 通义千问 VL / GPT-4o | 多模态大模型识别衣物属性 |
| **AI - 搭配推荐** | DeepSeek / GPT-4o | LLM 生成搭配方案 |
| **AI - 图片合成** | 预留（IDM-VTON 等） | MVP 阶段降级为参考图方案 |
| **天气 API** | 和风天气 API | 国内数据准，免费额度够 |

## 四、数据模型设计

### 4.1 核心数据表

```sql
-- 用户表
CREATE TABLE users (
    id            BIGSERIAL PRIMARY KEY,
    openid        VARCHAR(128) UNIQUE NOT NULL,  -- 微信 openid
    nickname      VARCHAR(64),
    avatar_url    TEXT,
    gender        SMALLINT DEFAULT 0,            -- 0未知 1男 2女
    height_cm     SMALLINT,
    weight_kg     SMALLINT,
    city          VARCHAR(64),
    style_tags    JSONB DEFAULT '[]',            -- 风格偏好标签
    membership    SMALLINT DEFAULT 0,            -- 0免费 1月卡 2季卡 3年卡
    member_expire TIMESTAMP,
    created_at    TIMESTAMP DEFAULT NOW(),
    updated_at    TIMESTAMP DEFAULT NOW()
);

-- 衣物表
CREATE TABLE clothing_items (
    id            BIGSERIAL PRIMARY KEY,
    user_id       BIGINT REFERENCES users(id),
    image_url     TEXT NOT NULL,                  -- 原图 URL
    thumbnail_url TEXT,                           -- 抠图后缩略图
    category      VARCHAR(32) NOT NULL,           -- 类型：top/bottom/dress/outerwear/shoes/bag/accessory
    sub_category  VARCHAR(32),                    -- 子类型：T恤/衬衫/卫衣…
    primary_color VARCHAR(32),                    -- 主色调
    colors        JSONB DEFAULT '[]',             -- 所有颜色
    style_tags    JSONB DEFAULT '[]',             -- 风格标签
    seasons       JSONB DEFAULT '[]',             -- 适合季节
    material      VARCHAR(64),                    -- 材质
    brand         VARCHAR(64),
    is_hidden     BOOLEAN DEFAULT FALSE,          -- 临时屏蔽
    wear_count    INT DEFAULT 0,                  -- 穿着次数
    embedding     vector(512),                    -- 图片向量（用于相似度检索）
    ai_raw        JSONB,                          -- AI 原始识别结果
    created_at    TIMESTAMP DEFAULT NOW(),
    updated_at    TIMESTAMP DEFAULT NOW()
);

-- 搭配方案表
CREATE TABLE outfits (
    id            BIGSERIAL PRIMARY KEY,
    user_id       BIGINT REFERENCES users(id),
    item_ids      BIGINT[] NOT NULL,              -- 包含的衣物 ID 列表
    occasion      VARCHAR(32),                    -- 场合
    weather       VARCHAR(32),                    -- 天气
    temperature   SMALLINT,                       -- 温度
    style_tags    JSONB DEFAULT '[]',
    ai_reason     TEXT,                           -- AI 搭配理由
    reference_img TEXT,                           -- 参考穿搭图 URL
    is_favorite   BOOLEAN DEFAULT FALSE,
    user_rating   SMALLINT,                       -- 用户评分 1-5
    is_public     BOOLEAN DEFAULT FALSE,          -- 是否公开
    created_at    TIMESTAMP DEFAULT NOW()
);

-- 搭配反馈表（用于优化推荐）
CREATE TABLE outfit_feedback (
    id            BIGSERIAL PRIMARY KEY,
    user_id       BIGINT REFERENCES users(id),
    outfit_id     BIGINT REFERENCES outfits(id),
    action        VARCHAR(16) NOT NULL,           -- like / dislike / save / share
    created_at    TIMESTAMP DEFAULT NOW()
);

-- 商品推荐表
CREATE TABLE product_recommendations (
    id            BIGSERIAL PRIMARY KEY,
    outfit_id     BIGINT REFERENCES outfits(id),
    category      VARCHAR(32),                    -- 推荐补充的品类
    product_name  VARCHAR(128),
    product_img   TEXT,
    product_url   TEXT,
    price         DECIMAL(10,2),
    source        VARCHAR(32),                    -- 来源：taobao/jd/xhs…
    commission    DECIMAL(5,4),                   -- 佣金比例
    created_at    TIMESTAMP DEFAULT NOW()
);
```

## 五、AI 能力设计

### 5.1 衣物识别 Pipeline

```
用户上传图片
    │
    ▼
┌──────────────┐
│ 图片预处理     │  → 压缩、校正、去背景（rembg 库）
└──────┬───────┘
       ▼
┌──────────────┐
│ 多模态 LLM    │  → 通义千问-VL / GPT-4o
│ 识别衣物属性   │     Prompt: "识别图中衣物的类型、颜色、风格、季节、材质"
└──────┬───────┘
       ▼
┌──────────────┐
│ 结构化输出     │  → JSON Schema 约束输出格式
│ + 校验修正    │
└──────┬───────┘
       ▼
┌──────────────┐
│ 生成 Embedding│  → CLIP 模型生成图片向量
└──────┬───────┘
       ▼
   存入数据库
```

### 5.2 搭配推荐 Prompt 设计思路

```python
OUTFIT_RECOMMEND_PROMPT = """
你是一位专业的时尚搭配师，精通小红书流行的穿搭风格。

## 用户信息
- 性别：{gender}
- 身高/体重：{height}cm / {weight}kg
- 风格偏好：{style_preferences}
- 所在城市：{city}

## 今日条件
- 天气：{weather}
- 温度：{temperature}°C
- 场合：{occasion}

## 用户衣橱（可用衣物）
{wardrobe_items}

## 任务
请从用户衣橱中挑选衣物，生成 3 套搭配方案。每套方案需要：
1. 选择具体的衣物（引用衣物 ID）
2. 说明搭配理由（为什么这几件搭在一起好看）
3. 给出风格标签
4. 说明适合的场景
5. 如果衣橱里缺关键单品，指出缺什么

## 搭配原则
- 颜色协调（同色系/对比色/万能黑白灰）
- 风格统一（不要混搭冲突风格）
- 符合天气和场合
- 考虑用户身材特点
- 参考当下小红书流行趋势

请以 JSON 格式输出。
"""
```

### 5.3 技术降级策略（MVP 阶段）

| 理想方案 | MVP 降级方案 | 说明 |
|---------|------------|------|
| AI 虚拟试穿合成 | 衣物拼图卡片 | 将选中衣物排列组合成搭配卡片 |
| 实时上身效果 | 匹配相似风格的小红书参考图 | 从图库中检索相似搭配的真人穿搭图 |
| 精准商品推荐 | 关键词推荐 + 手动运营 | 先用品类关键词匹配，后期接入 CPS 平台 |

## 七、后端 API 设计（核心接口）

### 7.1 用户模块

```
POST   /api/v1/auth/wx-login          # 微信登录
GET    /api/v1/user/profile            # 获取用户资料
PUT    /api/v1/user/profile            # 更新用户资料
PUT    /api/v1/user/style-preferences  # 更新风格偏好
```

### 7.2 衣橱模块

```
POST   /api/v1/wardrobe/items                # 上传衣物（图片 + AI 识别）
GET    /api/v1/wardrobe/items                 # 获取衣橱列表（支持筛选/分页）
GET    /api/v1/wardrobe/items/{id}            # 获取衣物详情
PUT    /api/v1/wardrobe/items/{id}            # 编辑衣物信息
DELETE /api/v1/wardrobe/items/{id}            # 删除衣物
PUT    /api/v1/wardrobe/items/{id}/hide       # 屏蔽/取消屏蔽衣物
GET    /api/v1/wardrobe/stats                 # 衣橱统计数据
```

### 7.3 搭配模块

```
POST   /api/v1/outfits/recommend              # 请求 AI 搭配推荐

GET    /api/v1/outfits/{id}                    # 搭配详情
POST   /api/v1/outfits/{id}/feedback           # 搭配反馈（喜欢/不喜欢）

```

### 7.4 种草 & 分享

```
GET    /api/v1/recommendations/missing-items   # 衣橱缺失单品分析
GET    /api/v1/recommendations/products        # 商品推荐列表
POST   /api/v1/share/card                      # 生成分享卡片
```
